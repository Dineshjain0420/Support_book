GitHub Code Search Syntax Guide

This guide helps you understand how to build search queries for GitHub's code search feature.

About Code Search Query Structure

Search queries consist of search terms and qualifiers to narrow down the search. Here's an example of a basic query:

http-push

This will match files containing the term "http-push" or files named with that term in their path.

Query for an Exact Match

You can search for an exact string by surrounding it with quotes:

"sparse index"

This will match any file containing the exact phrase "sparse index".

Searching for Quotes and Backslashes

To search for quotes or backslashes in code, use escape sequences:

For quotes: "name = \"tensorflow\""

For backslashes: \\

Using Boolean Operations

GitHub Code Search supports boolean operations such as AND, OR, and NOT.

Example:

language:python AND "function" OR "method"

This query searches for Python files that contain either the term "function" or "method".

Using Qualifiers

Qualifiers narrow down your search results. For example:

  • Repository qualifier: repo:github-linguist/linguist
  • Language qualifier: language:ruby
  • Path qualifier: path:unit_tests

These allow you to search within specific repositories, languages, or file paths.

Using Regular Expressions

GitHub Code Search supports regular expressions. For example, to search for the regular expression sparse.*index:

/sparse.*index/

This matches occurrences of the word "sparse" followed by any number of characters, then the word "index".

Case Sensitivity

By default, searches are case-insensitive. To perform a case-sensitive search, use a regular expression with case insensitivity turned off:

/(?-i)True/

For more information, visit GitHub Search Documentation.
